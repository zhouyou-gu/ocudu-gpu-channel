- The patched OCUDU ZMQ Docker rebuild compiles UHD/OCUDU from source and is slow; a clean rebuild can take many minutes before the gNB reports `zmq` as an available radio type.
- The CUDA backend supports integer/fractional delay only as a chain-leading step (applied host-side at staging, with a cross-batch host delay line); a non-leading delay still fails fast on `backend: cuda` and stays CPU-only. Gain, path loss, phase, CFO, AWGN, and a leading delay are GPU-supported.
- Broker hot path no longer allocates poll vectors or TX receive IQ buffers in each loop, but still constructs link-key strings and uses unordered-map lookups while processing RX requests.
- CUDA MVP still does a `cudaStreamSynchronize` at the end of every `process_into` call, so batches within one link are not pipelined; with `process_mutex` removed the two directions' per-link streams do now overlap, but cross-batch pipelining within a link remains future work.
