- Remote active network path is Wi-Fi (`wlp129s0`), which is suitable for SSH/control but not accepted as a real-time distributed IQ data path.

The only non-fatal signal left is an occasional `rx_starvations` (host scheduling on the shared box), now treated as soft.
- `scripts/remote/ocudu-attach-smoke.sh` previously hung on teardown when srsUE wedged on shutdown; fixed to `docker rm -f` the srsUE container before `wait`.
- The 614400-sample CUDA interop benchmark is red under the current slot-duration gate even though the batch spans about 26.7 ms of IQ at 23.04 MS/s; the benchmark gate needs a separate interop-batch interpretation before publishing results.
- The patched OCUDU ZMQ Docker rebuild compiles UHD/OCUDU from source and is slow; a clean rebuild can take many minutes before the gNB reports `zmq` as an available radio type.
- The CUDA backend supports integer/fractional delay only as a chain-leading step (applied host-side at staging, with a cross-batch host delay line); a non-leading delay still fails fast on `backend: cuda` and stays CPU-only. Gain, path loss, phase, CFO, AWGN, and a leading delay are GPU-supported.
- Broker hot path no longer allocates poll vectors, TX receive IQ buffers, or constructs link-key strings in the per-slot path -- the latter was fixed in the Phase 2 D2b commit.
- CUDA backend still does a `cudaStreamSynchronize` at the end of every `process_superposition` call, so batches within one link are not pipelined; with `process_mutex` removed the two directions' per-link streams do now overlap, but cross-batch pipelining within a link remains optional future work (deferred with Phase 2 D4 -- PCIe is no longer binding).
